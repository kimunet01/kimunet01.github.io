#### 2-1 용어 정리
`용어 정리만 하고 간단하게 넘어 갈 예정이다.`
```
모델링 : 장면(Scene) 내부의 물체(Object)를 정의 하는 작업이다.
선분의 양 끝점, 다각형의 정점의 위치를 명시하는 작업.

렌더링 : 모델링으로 정의된 물체를 어떻게 그릴지에 관련된 것.

래스터 : 그림의 기본 요소인 화소를 의미하며 Pixel이다.

인점 : 컬러 모니터에서 RGB 색을 띤 한 점 -> 현대의 서브픽셀

도트 피치 : 동일 색상의 인점 사이의 거리 -> 현대의 픽셀 피치

주사선 : 화면의 가로 방향 화소를 따라 진행하는 선
```

#### 2-2 래스터 그래픽 장치

✅ `래스터 장치`

```
래스터 장치는 화소 단위로 디스플레이 된다.
하나의 화소 내부 색은 모두 동일한 밝기로 간주한다.
```

✅ `인터레이싱`

```
과거에 화면 전체를 다 그리기에 데이터 전송 속도가 너무 느렸기에
화면을 홀수 줄과 짝수 줄을 번갈아 그린 방식.
초당 30번 재생이 요구될 때, 인터레이싱 방식은 반쪽 화면을 60번 주사한다.
해당 방식으로 대역폭을 절약하였다.
```

✅ `래스터 장치 기본 구조`

```
래스터 장치에서의 화면은 사각형 모양의 화소 단위로 표현된다.
하나의 화소는 일정한 크기를 가지기 때문에, 화소 크기 이하로 분할하여 다른 색으로 밝힐 수 없다.
사각형 모양으로 이루어져 물체를 나타내기 때문에 경계선 부분이 거칠에 보이게 되는데
이러한 현상을 에일리어스(계단현상) 이라고 한다.
```

✅ `에일리어스`

```
연속적인 신호를 이산적인 화소 단위로 변환하는 과정에서 발생하는 정보의 손실로 인하여
거칠게 보이게 되는 것이다.
```

✅ `스캔 변환 / 래스터 변환`

```
선분의 양 끝 점의 좌표가 있을 때, 그 점의 위치를 수학 적 표현으로부터 화면 화소 단위로
변한하는 과정을 스캔 변환 또는 래스터 변환이라고 한다.
스캔 변환 알고리즘은 해당 경우에 어떤 화소를 얼마나 밝힐지에 관련된 것이다.
```

✅ `프레임 버퍼`

```
그래픽 프로세서에 내장된 메모리로, 화소당 밝기를 저장하는 곳이다.
프레임 버퍼의 용량은 색상의 종류와 연관되어 있다.
예를 들어 화소당 1비트를 할애 할 경우 흑/백 정보만 담을 수 있다.

현대의 프레임 버퍼는 RGBA 각 8비트씩 해서 32비트 방식이다.
화소당 32비트의 정보를 가지고 있는 셈이다.

프레임 버퍼의 용량은 해상도와도 관련이 있다.
1920x1080 해상도의 경우 1920x1080개의 화소를 가지고 있는데, 해당 개수에
화소당 32비트를 할애하면 프레임 버퍼의 용량은 1920x1080x32비트 이다.

따라서 현대의 교해상도, 고발광 환경일수록 GPU의 VRAM 요구량이 엄청나게 늘어난다.
```

✅ `DAC`
```
프레임 버퍼의 내용을 화면에 그려넣는 것은 비디오 컨트롤러의 역할.
비디오 컨트롤러 내부에 DAC가 내장되어 있다.
화소를 주사할 때 프레임 버퍼의 디지털 비트 값에 비례하는 아날로그 전압으로 변환시켜
해당 화소의 밝기를 조절한다.
```

#### 2-3 벡터 그래픽 장치

✅ `벡터 그래픽 장치`
```
해당 방식은 래스터 장치처럼 화소 단위로 밝혀지는것이 아니라 전자총이 닿는 부분마다
밝혀지게 되는 방식이다.

하지만 현대에는 CRT 모니터가 사라져 전자총처럼 자유롭게 할 수 없고,
그래픽 처리속도의 향상과 , 저가 메모리의 출현으로 사라지게 되었다.

하지만 CPU나 GPU가 (0,0)에서 (10,10) 점을 이으라는 벡터 명령을 내렸을 때,
GPU 내부의 래스터라이저가 이 선을 지나가는 픽셀들을
계산하여 계산된 픽셀들에만 RGB 값을 채워넣는 방식의 논리적인 벡터 연산은 남아있다.

물론 물리적인 벡터 장치가 쓰이는 특수 분야가 있다고는 한다.
하지만 현대 모니터는 벡터 방식을 포기하였다 !
```

✅ `벡터 그래픽 장치의 Pros and Cons`
```
⭕ 장점
- 에일리어스가 발생하지 않는다.
- 래스터 변환이 불필요하다.
- 소용량의 디스플레이 리스트를 사용한다.
    ㄴ 실제 픽셀 값을 저장하지않고 수학적 정보만 저장하기 때문.

❌ 단점
- 렌더링이 느리다.
    ㄴ 전자총이 선을 따라가며 그리기 때문.
```

#### 2-4 그래픽 출력 장치

✅ `LCD`
```
액체와 고체의 중간 형태인 액정이 지닌 전기적, 광학적 특성을 이용한 모니터이다.
액정은 외부에서 가하는 전압에 따라 누워있거나 서있는 상태가 변화하기 때문에 이 점을 이용한다.
 
LCD 화소에는 좌우에 편광기가 있다. 편광기는 특정 방향으로 진동하는 빛만 통과시키는데,
편광기를 통과한 빛이 서브픽셀마다 배치된 액정이 받은 전기 신호에 따라 빛을 얼마나 통과시킬지 결정.
(누워 있거나 서있는 상태를 이용해 빛의 방향을 변경)
이후에 컬러 필터를 통과하며 색이 입혀지는 방식이다.

CRT에 비해 가볍고 완전 평면이 가능하며, 유해 전자파가 없지만
밝기와 명암대비가 약하고 색상이 부자연스럽다.
```

✅ `PDP`
```
플라즈마 상태를 사용한 디스플레이로,
네온이나 크세논 같은 가스를 채워넣어 전압을 걸어 플라즈마 상태로 만들어
방출된 자외선이 픽셀 내벽의 RGB 형광체에 부딪혀 색을 내는 방식이다.

LCD에 비해 밝기와 명암대비가 좋았지만 픽셀크기를 줄이기 어려웠고,
정지된 화면을 오래켜두면 형광체가 타버려 잔상이 영구적으로 남고,
전기를 사용한다는 점에서 단점이 있었다.
```

`이외에 LED와 OLED도 있지만 넘어가겠음`

✅ `입체 영상`
```
두 눈의 거리차이로 입체감을 느끼는 현상인 양안시차를 사용한다.
두 개의 평면 영상을 만든 후 편광 안경을 사용해 좌,우 같지만 다른 영상을 보게 하는 방식이다.

안경을 사용하지 않고 렌즈 배열 방식의 모니터를 사용해 입체 영상을 보는 방법도 있다.
화면 표면에 반구형의 렌즈를 붙여 왼쪽과 오른쪽 영상이 서로 다른 방향으로 진행하도록 하여
왼쪽 영상이 왼쪽 눈에, 오른쪽 영상이 오른쪽 눈에 들어오게 한다.
이 방식은 옛날에 자주봤던 3D 책갈피에 적용되는 방식과 유사하다.
```

✅ `홀로그래피`
```
해당 방식은 빛의 간섭과 회절 속성, 레이저, 육각렌즈를 사용해 3차원 물체의 모습을 공간에
투사하는 방식이다.
```

#### 2-5 그래픽 프로세서

✅ `GPU의 등장`
```
초창기 그래픽 처리는 모두 CPU가 했지만, 그 부담을 덜어주기 위하여 GPU가 등장하였다.

GPU는 단순한 다량의 산술 연산을 병렬적으로 처리할수있는 장치이다.
```

✅ `그래픽 카드의 구조`

![그래픽 카드 구조](/assets/img/posts/2026-03-24-CG/Graphic_Card_architect.png)
`출처 : 주우석 저자, 'OpenGL로 배우는 3차원 컴퓨터 그래픽스' 60page, 그림 2-26`

```
그래픽 처리기 (GPU) : 디스플레이 프로세서 + 지오메트리 프로세서

지오메트리 프로세서 : 3차원 물체에서 다각형을 구성하는 정점을 기준으로 물체의
이동, 회전, 절단, 투상, 조명, 텍스처 등의 다향한 그래픽 처리 작업을 담당.

디스플레이 프로세서 : 비트맵 영상 이동/색칠, 윈도우 위치/크기 조절, 선분 그리기 등
2차원 영상처리와 관련된 연산을 담당.

VRAM : 그래픽 카드에 들어있는 물리적인 메모리 칩으로 3D 모델 데이터, 텍스처, 
쉐이더 프로그램, 깊이정보 등의 정보가 저장된다. 프레임 버퍼도 VRAM에 포함.

프레임 버퍼 : 프론트 버퍼와 백 버퍼 2중 포트 형태로,
프론트 버퍼는 현재 모니터에 실시간으로 화면을 전송한다.
백 버퍼는 다음 프레임을 그리는 작업을 한다.
백 버퍼가 다음 프레임을 다 그리게 되면 프론트 버퍼와 스왑하여 역할을 서로 바꾼다.
깜빡임과 찢어짐을 개선하기 위해 2중구조로 사용한다.

비디오 컨트롤러 : 프론트 버퍼를 읽어 데이터를 모니터로 전송하는 역할을 한다.
프레임 버퍼에 있는 0,1 데이터를 모니터가 인식할 전압 신호로 변환한다.
```

✅ `이중 버퍼의 한계`
```
이중 버퍼링만으로는 티어링(찢어짐) 현상을 완벽하게 개선하지 못한다.
백 버퍼가 모니터가 한 프레임을 출력 하기 전에 작업을 마쳐 교체 하게 되면
찢어짐이 발생한 기준으로 상단은 이전 프레임, 하단은 새로 갱신된 프레임이 그려지게 된다.
```
![모니터 티어링 예시](/assets/img/posts/2026-03-24-CG/Mornitor_Tearing.png)
`출처 : https://en.wikipedia.org/wiki/Screen_tearing`

✅ `수직 동기화`
```
이중 버퍼의 티어링 현상을 해결하기 위해 도입된 방식이다.
백 버퍼에 다음 프레임이 모두 그려져도 모니터가 한 프레임을 출력 하기 전에는
스왑을 하지 않는 방식이다.

하지만 GPU가 성능이 좋아도 모니터의 속도에 맞춰야 하기 때문에 인풋 랙이 발생하고,
GPU가 성능이 조금 안좋을 경우에는 모니터가 현재 프레임을 이전 프레임으로 보여주기 때문에
프레임이 떨어지는 문제가 발생 할 수 있다.
```

✅ `삼중 버퍼링`
```
이 문제들을 개선하기 위해서 버퍼를 세개를 사용하는 것이다.

프론트 버퍼, 백 버퍼 1, 백 버퍼 2 의 구조로
백 버퍼 1에 그림을 다 그렸는데 프론트 버퍼가 아직 모니터에 출력 하고 있다면
다음 프레임을 백 버퍼 2가 그리기 시작한다.
만약 모니터가 한 프레임을 출력하기 전에 백 버퍼 1,2 모두 렌더링이 되었다면
가장 최근에 그린 장면으로 Swap한다. 출력되지 못한 중간 프레임은 폐기 된다.

GPU의 유휴 시간이 줄어들어 프레임 방어에 좋고, 최근 장면을 보여주기 때문에 인풋 렉도 줄어든다.
```
